# HomeCleaning-BookingSystem

# 家政保洁预约小程序与后台管理系统 - 需求分析文档

## 一、 C端：用户端（微信小程序）

C端依托微信生态，侧重于提供便捷的上门服务预约、清晰的费用明细、顺畅的支付体验以及透明的退款机制。

### 1. 账户与基础功能
* **授权登录：** 采用微信一键授权登录，静默建立用户档案。
* **个人中心：** 查看历史订单、退款记录。
* **地址管理 (新增)：** 用户可添加、修改、删除服务地址（包含联系人、手机号、省市区、详细门牌号等），下单时需选择对应地址。

### 2. 服务浏览与预约配置 (核心)
* **服务项目列表：** 展示保洁项目（如：日常保洁、深度保洁、家电清洗等），标明计价方式（按小时/按面积/按件）及图文介绍。
* **档期与运力选择：** * **日历视图：** 展示近期可预约的日期。
    * **时段与运力容量：** 用户选择具体日期后，展示该日期的细分时段（如 09:00-11:00）及其**剩余运力（即该时段空闲的保洁员数量）**。
* **订单信息录入：** 确认服务地址、选择服务时长/数量，填写特殊备注（如：家里有宠物、需自带某种清洁剂等）。

### 3. 订单与支付
* **创单锁定：** 提交预约后生成待支付订单，并**临时锁定该时段运力**（建议锁定 10-15 分钟）。
* **支付网关：** 接入微信支付，直接调用小程序官方支付 API。
* **状态流转：** 微信支付成功回调后，正式扣减运力库存，订单状态变更为“已支付（待派单/待上门）”。超时未支付则订单自动取消，释放运力。

### 4. 自助取消与退款 (核心)
* **取消预约入口：** 在订单详情页提供“取消预约并退款”按钮。
* **阶梯退款试算：** 用户点击取消时，系统根据当前时间与预约上门时间的差值，调用后端的阶梯退款规则引擎。**实时计算并展示**违约金及实际可退金额（例如：提前24小时全额退，提前2-24小时扣除20%违约金，2小时内不可退等）。
* **自动化退款：** 用户确认后，系统调用微信原路退款接口，订单扭转为“已退款/部分退款”，同时**释放该订单占用的运力**。

## 二、 B端：管理后台系统

B端侧重于保洁项目维护、运力排班配置、动态规则定义和业务履约跟进。

### 1. 运营工作台
* **数据看板：** 核心指标监控，包括今日上门订单数、剩余运力、预计营业额、退款笔数及金额。

### 2. 资源与运力管理 (核心)
* **保洁项目管理：** 维护服务名称、基础价格、封面图、详情描述、上架状态。
* **排期与运力配置：** * 支持按天或按周批量生成排期。
    * 配置指定日期、指定时段的**最大可预约运力上限（保洁员数量）**。

### 3. 动态退款规则配置 (核心)
* **规则引擎设置：** 提供可视化的阶梯违约金配置界面。定义不同时间节点（距离上门时间的倒计时）对应的违约金扣除比例。
* **规则绑定：** 可将不同的退款规则绑定到不同的保洁项目上。

### 4. 订单与履约管理
* **预约看板：** 直观展示每天各时段的预约服务明细（含上门地址、客户备注），便于线下调度保洁员。
* **综合订单检索：** 支持按手机号、订单状态、服务日期进行查询。
* **人工干预退款：** 针对客诉，管理员拥有最高权限，可绕过自动规则手动发起全额或部分退款。
* **核销管理 (履约)：** 保洁员上门完成服务后，管理员在后台**手动将订单状态核销为“已完成”**。

## 三、 技术架构与落地建议

1. **前后端技术选型：**
    * **后端 API：** 采用 C# 基于 **.NET 8** 构建高性能 RESTful API。引入 **ABP Framework** 快速搭建底层基础设施，利用其自带的模块化、权限控制和工作单元机制加速业务开发。
    * **数据持久化：** 使用 **EF Core** 配合 **SQL Server 2016** 进行数据存储与实体映射。
    * **前端呈现：** B端管理后台使用 **Vue.js 3** 搭建单页应用；C端微信小程序可采用基于 Vue 语法的 Uni-app 框架，实现技术栈的平滑过渡。
2. **并发与防超卖机制：**
    * 热门节假日家政预约易产生并发。针对运力扣减，建议在 SQL Server 层面利用 EF Core 的并发令牌（如 RowVersion 乐观锁），或结合缓存技术防止运力超发。
3. **退款规则快照设计：**
    * 用户下单时，将当前的“退款规则”以 JSON 格式**硬拷贝（Snapshot）至订单记录表**中。未来计算违约金严格依据快照执行，避免后台规则变更引发历史订单客诉。
4. **后台任务调度：**
    * 借助后台工作者服务（Background Workers）处理超时未支付订单的自动取消与运力释放。
