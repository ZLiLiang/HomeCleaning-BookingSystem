# 家政保洁预约系统 - 详细开发落地计划

以下是基于需求分析文档整理的系统完整开发与落地计划，各阶段按照技术依赖与业务逻辑先后顺序进行拆解。

## 阶段一：基础设施与架构搭建

本阶段旨在完成前后端基础工程的初始化，搭建符合技术选型规范的底层框架结构。

- [ ] **1.1 后端服务初始化**
  - 构建基于 **.NET 8** 平台的 RESTful API 基础工程。
  - 全面引入 **ABP Framework**，配置模块化（Modularity）、依赖注入（DI）与工作单元（Unit of Work）机制。
  - 建立基于角色的权限控制体系（RBAC）与审计日志底层支持。
- [ ] **1.2 数据库与持久化配置**
  - 部署 **SQL Server 2016** 作为核心数据库。
  - 集成 **EF Core**，完成基础实体映射配置。
  - 规范数据访问层架构，统一使用 **LINQ to Data** 进行强类型查询。
- [ ] **1.3 前端工程搭建**
  - **B端管理后台**：基于 **Vue.js 3** 结合 Element Plus 或 Ant Design Vue 初始化单页应用（SPA）工程。
  - **C端用户端**：采用 **Uni-app**（Vue 语法）或原生技术初始化微信小程序工程。

---

## 阶段二：领域驱动设计与后台基础（B端侧重）

本阶段重点完成管理后台的核心业务支撑模块，为 C 端提供必要的数据字典与规则基础。

- [ ] **2.1 保洁项目管理模块**
  - 开发服务项目的维护（增删改查），包括服务名称、计价方式（按小时/面积/件）、封面详情及上下架状态。
- [ ] **2.2 排期与运力配置系统**
  - 实现按天或按周批量生成保洁排期数据。
  - 提供日历视图，配置指定日期与时段的最大可预约运力（保洁员数量上限）。
- [ ] **2.3 动态退款规则引擎**
  - 开发可视化的退款规则配置界面，定义多级阶梯时间节点的违约金扣除比例。
  - 实现退款规则与具体保洁项目的动态绑定逻辑。
- [ ] **2.4 营销与卡券管理台**
  - 构建包含满减、折扣、无门槛券在内的卡券模板库。
  - 配置复杂的发券规则（适用项目限制、动态有效期）与发放机制（定向派发、主动领取、系统触发）。
  - 建立后台优惠券发放、领取与核销的流转台账。

---

## 阶段三：C 端小程序核心交互开发

实现面向用户的完整业务体验流，确保前端交互顺畅且数据呈现准确。

- [ ] **3.1 账户与用户中心**
  - 接入微信一键授权登录机制，静默创建用户档案。
  - 开发“我的”模块，包含历史订单展示、退款记录查询。
  - 开发多状态（未使用/已使用/已过期）优惠券卡包与多层级地址管理功能。
- [ ] **3.2 服务浏览与预约档期**
  - 首页及列表页展示保洁项目及营销“领券中心”入口。
  - 交互式日历开发，展示日期、细分时段（如09:00-11:00）及对应的实时空闲运力。
- [ ] **3.3 订单创建与结算**
  - 收集预约上门地址、时长/数量及客户特殊备注。
  - 结算引擎开发：系统自动匹配并勾选最优优惠券，支持手动切换，并实时动态试算服务总价、抵扣金与最终应付款。

## 阶段四：核心复杂业务流转与机制落地

本阶段处理系统中难度较高的并发、支付回调、异步任务及财务数据一致性问题。

- [ ] **4.1 高并发防超卖与运力扣减**
  - 实现下单临时锁定运力机制（10-15分钟）。
  - 在 SQL Server 层面运用 **EF Core 并发令牌（RowVersion乐观锁）** 处理运力与限量券的并发冲突。
  - 架构中预留 Redis 接口，利用 Lua 脚本支持极端大促场景下的高并发库存预扣减。
- [ ] **4.2 财务级快照（Snapshot）机制**
  - 下单支付瞬间，将当期的退款规则、商品原价、优惠券明细序列化为 JSON。
  - 将 JSON 硬拷贝至订单明细表持久化，彻底隔离后续后台全局规则变更对已成单历史数据的影响。
- [ ] **4.3 后台异步任务调度（Background Jobs）**
  - 依托 ABP 的后台工作者服务（或 Quartz.NET / Hangfire），实现订单超时未支付的自动取消、运力释放及优惠券退回。
  - 实现每日凌晨执行的优惠券过期状态批量巡检更新任务。
  - 实现距上门前 2 小时自动触发的微信小程序服务提醒订阅消息。
- [ ] **4.4 微信支付与自动化退款引擎**
  - 深度接入小程序官方支付 API，实现成功回调后的库存实质扣减与状态扭转。
  - 开发订单详情页的自主退款入口，实时调用阶梯退款规则计算违约金及实际退款额。
  - 接入微信原路退款接口，执行资金自动化结算，并依规则退回或作废关联优惠券。

## 阶段五：履约闭环与数据监控看板

完善平台后续的订单履约管理与运营数据支撑。

- [ ] **5.1 订单履约与核销**
  - 开发 B 端综合订单检索（支持多条件筛选）与每日预约看板，辅助线下人力调度。
  - 引入人工干预机制：赋予超管绕过自动阶梯规则进行全额/部分退款的客诉处理权限。
  - 业务核销闭环：服务完成后，管理员在后台经 ABP 工作单元保障事务一致性，手动将订单核销为“已完成”，触发后续平台结算。
- [ ] **5.2 数据可视化工作台**
  - 构建实时数据看板，监控今日上门单量、剩余运力、预计营业额及退款风控数据。
  - 开发营销专属看板，追踪卡券核销数量与让利抵扣总金额等核心指标。
